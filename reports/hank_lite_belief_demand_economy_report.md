# LLM Beliefs in a HANK-Lite Demand Economy

## Bottom Line

The strongest result so far is a bridge result: when prior GPT-5.5 household belief payloads are replayed through the upgraded HANK-lite demand economy, the system clears the current macro behavior/dynamics scorecard. The micro behavior gate passes, the qualitative impulse-response gate passes, and the vintage out-of-sample gate remains open because the date-free vintage cards, hidden targets, and control forecasts have not yet been built.

That is a narrower claim than "LLM agents simulate the economy." It is also a more useful one. The evidence says an LLM can act as a household belief engine inside a constrained macro system: the model updates beliefs; the structural economy enforces budgets, accounting, aggregation, feedback, and shock response.

## What Was Built

The economy is deliberately small and auditable:

- 12 weighted household cells with different income, liquidity, job-risk, age, asset, debt, and baseline MPC profiles.
- A belief module that produces household expectations for inflation, income growth, job loss risk, confidence, attention, and precautionary saving.
- A structural consumption layer that converts beliefs into feasible household behavior.
- Explicit transfer allocation across consumption, debt repayment, and liquid saving.
- Debt-stock accounting and liquid-buffer carryover across periods.
- A closed-loop demand environment where consumption feeds output, employment, inflation, and a Taylor-rule policy rate.
- Five scenarios: baseline, transfer shock, rate hike, job-risk shock, and belief-feedback amplification.

The key design choice is separation of roles. The LLM supplies beliefs. The code owns feasibility.

The current scorecard was generated by replaying the prior GPT-5.5 belief payloads through the upgraded mechanism layer with zero new LLM calls. The replay run produces `hank_lite_belief_lab_ready`; the macro-validity scorecard produces `macro_behavior_dynamics_ready_but_vintage_oos_unscored`.

## What Changed

The earlier live run showed that GPT-5.5 beliefs produced plausible household heterogeneity and macro feedback. The latest mechanism pass tightened the behavior layer. It added:

- transfer windfall allocation across spending, debt repayment, and liquid saving;
- debt repayment and debt-stock accounting;
- liquid-buffer relief that carries forward and decays;
- gradual rate pass-through for monetary shocks;
- a macro-validity scorecard that separates micro behavior, impulse-response shape, and vintage OOS readiness.

This moves the project from "the beliefs look good" to "the beliefs produce plausible behavior when embedded in a constrained economy."

## Provenance And Scope

The current result is a replayed-belief result. The source GPT-5.5 demand run contains 100 LLM belief records across 5 scenarios and 20 periods; 38 were fresh live calls and 62 were cache hits from that run's fresh cache. The current replay spends zero new LLM calls and reuses those belief payloads while rerunning the deterministic structural economy.

That makes the result reproducible and cheap to audit, but it leaves two experimental questions for the next pass. First, the source beliefs were generated before the mechanism upgrade, so some prompt-state variables differ from the upgraded replay path. Second, broader macro validity still needs a hidden-outcome vintage test rather than a readiness check.

The scope is therefore:

> GPT-5.5 belief payloads, embedded in the upgraded HANK-lite demand economy, clear the current behavior and qualitative dynamics bridge. Full macro validity requires a clean fresh-belief run and scored vintage out-of-sample performance.

## Next Experimental Setup

The next experiment should do two things at once: remove the replay-provenance weakness and create the first true out-of-sample macro behavior test.

1. Run a clean fresh-belief mechanism pass.
   Use the upgraded economy as the prompt/state source, generate fresh belief payloads, and store prompt/state hashes with each raw record. This tells us whether the result survives when the LLM sees the current mechanism state rather than an earlier period-indexed state.

2. Build the date-free vintage OOS runner.
   Convert the FRED/ALFRED vintage panel into date-free cards, map as-of macro conditions into the demand economy, withhold future consumption/saving/labor outcomes, and score model forecasts against no-change, rolling/trend, and SPF-style controls.

3. Add comparative macro-validity gates.
   Keep the absolute pass/fail thresholds, but add direct comparisons against representative-agent, adaptive heterogeneous, and naive persona baselines. The next claim should say not only that the LLM belief economy behaves plausibly, but where it improves on simpler mechanisms.

4. Add calibration and sensitivity bands.
   Freeze the mechanism parameters, then perturb transfer size, rate-shock size, feedback gain, and household-cell composition. The result should survive moderate parameter movement rather than depend on one tuned setup.

If those gates clear, the claim becomes materially stronger: LLM belief modules do not merely improve macro expectations in isolation; they improve behavior and dynamics in a constrained economy, including on hidden out-of-sample macro states.
